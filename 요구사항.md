## 2. 로고 & 브랜드 타이포그래피 (★중요★)

앱 상단 로고는 **방송 프로그램 로고처럼** 디자인. 일반 텍스트로 렌더링하지 말고 **장식적 타이포그래피 컴포넌트**로 구현.

- **주 로고:** `오륜기파`
 - 굵은 세리프 또는 디스플레이 서체, K-pop 서바이벌 프로그램 타이틀 카드 느낌
 - 글자마다 약간의 기울기 / 음영 / 메탈릭 그라디언트 허용
 - 추천 폰트 후보: `Black Han Sans`, `Jua`, `Gmarket Sans Bold`, `Pretendard Black` 중 선택
- **부제:** `SEASON 2026` (영문 대문자 병기) 또는 `시즌 2026`
 - 주 로고보다 작게, 얇은 산세리프, 자간 넓게(letter-spacing: 0.2em~)
 - 양옆에 장식선(━ SEASON 2026 ━) 또는 별(★) 장식 허용
- **오륜기(五輪旗) 모티프:** 5개 컬러 원(파랑/노랑/검정/초록/빨강)을 로고 옆 또는 배경에 은은하게 배치 — 과하지 않게

홈 화면 최상단 히어로 영역에 이 로고가 시그니처처럼 박혀야 함.

## 4. 데이터 모델

### 4-1. Round (라운드)
```
{
  id: uuid,
  round_no: int (1~10),
  date: date (YYYY-MM-DD),
  course_name: string  // 골프장명
}
```

### 4-2. Score (스코어) — round 1개당 player 4명 행
```
{
  round_id: uuid (FK),
  player: enum('비비', '에이블', '에단', '제리'),
  total_strokes: int,   // 총타수 — 낮을수록 좋음
  total_putts: int,     // 총 퍼팅수 — 낮을수록 좋음
  birdies: int,         // 총 버디수 — 높을수록 좋음
  game_points: int      // 게임포인트(±, 음수 허용) — 높을수록 좋음
}
```

### 4-3. 집계 규칙 (자동 계산)
- **평균 총타수** = 해당 플레이어의 총타수 합 ÷ 참여 라운드 수 (소수 첫째자리)
- **평균 퍼팅수** = 동일 방식 (소수 첫째자리)
- **누적 버디수** = 단순 합 (정수)
- **누적 게임포인트** = 단순 합, 음수 가능 (정수, ± 부호 표시)
- **진행도** = `현재까지 입력된 라운드 수 / 10`

## 5. 화면 명세

### 5-1. 홈 화면 (메인 · 탭 1)

**구조 (위→아래 순서):**

**(A) 히어로 영역**
- `오륜기파` 로고 타이포그래피 (2장 규격 준수)
- 부제 `SEASON 2026`
- 진행도 뱃지: `6 / 10 ROUNDS` (남은 라운드 수 + 진행 프로그레스 바)

**(B) 누적 통계 표 — 최상단 배치 ★**

플레이어가 **세로축(행)**, 지표가 **가로축(열)**인 4행 × 4열 표:

| 플레이어 | 평균타수 ↓ | 평균퍼팅 ↓ | 누적버디 ↑ | 누적포인트 ↑ |
|---|---|---|---|---|
| 비비 | 84.3 | 31.2 | 5 | +12 |
| 에이블 | 87.1 | 32.0 | 3 | −4 |
| 에단 | 82.9 | 30.5 | 7 | +18 |
| 제리 | 89.4 | 33.1 | 2 | −6 |

- 헤더에 방향성 아이콘(↓ 낮을수록 좋음 / ↑ 높을수록 좋음) 표시
- 평균타수 오름차순 정렬 기본
- 플레이어 셀에는 고유 색상 닷(●) 또는 좌측 색상 바
- 1위 행(평균타수 최저)은 **왕관 아이콘 👑 + 골드 배경 하이라이트**
- 숫자는 Tabular figures(고정폭 숫자), 크고 굵게

**(C) 플레이어 카드 (4개, 세로 스택 또는 2×2 그리드)**

각 카드 구성:
- 상단 바: 플레이어 고유 색상 + 이름(비비/에이블/에단/제리) + 순위 뱃지(1~4위)
- 큰 숫자 4개: 평균타수, 평균퍼팅, 누적버디, 누적포인트
- 각 숫자 아래 작은 라벨
- 1위 카드는 테두리 강조 + 왕관 아이콘

**(D) 우상단 FAB / "+" 버튼 → 입력 화면 진입**

### 5-2. 기록 화면 (탭 2)

**최신 날짜 → 과거** 순 내림차순 리스트.

**한 라운드 카드 구조:**
```
┌───────────────────────────────────┐
│ R06 · 2026.09.14 · 레이크사이드CC  │
├──────┬──────┬──────┬──────┬──────┤
│      │ 타수 │ 퍼팅 │ 버디 │ 포인트│
├──────┼──────┼──────┼──────┼──────┤
│ 비비 │  82  │  30  │  2   │  +5  │
│ 에이블│  88  │  32  │  1   │  -2  │
│ 에단 │  80  │  29  │  3   │  +7  │
│ 제리 │  91  │  34  │  0   │  -10 │
└──────┴──────┴──────┴──────┴──────┘
```
- 행의 플레이어 이름 좌측에 고유 색상 바
- 해당 라운드 **최저 타수 셀**에 강조(굵게 + 배경 틴트)
- 카드 탭 → 수정/삭제 옵션 (확인 다이얼로그 필수)

---

### 5-3. 추이 화면 (탭 3)

**구조 (위→아래 순서 고정):**

**(A) 총타수 시계열 그래프**
- X축: 라운드 회차(R01~R10) 또는 날짜
- Y축: 총타수, **Y축 반전(낮은 값이 위로)** + 우상단 주석 `↓ 낮을수록 좋음`
- 4명 고유 색상 꺾은선, 포인트 마커, 범례 상단
- 포인트 탭시 툴팁: `날짜 · 골프장 · 해당 플레이어 타수`

**(B) 총타수 표 (그래프 바로 아래)**

세로 인덱스: `날짜 + 골프장명` (최신순), 가로 인덱스: 플레이어 이름

| 라운드 | 비비 | 에이블 | 에단 | 제리 |
|---|---|---|---|---|
| 2026.09.14 레이크사이드CC | 82 | 88 | 80 | 91 |
| 2026.08.02 이스트밸리CC | 85 | 86 | 83 | 89 |
| … | … | … | … | … |

- 각 행의 최저값 셀 강조
- 컬럼 헤더에 플레이어 고유 색상 닷

**(C) 총 퍼팅수 시계열 그래프** (A와 동일 구조, 지표만 교체)

**(D) 총 퍼팅수 표** (B와 동일 구조, 지표만 교체)

> **주의:** 이 순서(그래프→표→그래프→표)를 정확히 지킬 것. 버디/게임포인트는 이 화면에 포함하지 않음(요구사항 외).

### 5-4. 입력 화면 (라운드 추가)

진입: 홈 우상단 `+` 또는 FAB.

**필드 (위→아래):**
1. **회차** — 자동 감지(다음 미입력 회차), 드롭다운 수정 가능(1~10)
2. **날짜** — 기본값 오늘, 날짜 피커
3. **골프장명** — 텍스트 입력, 이전 입력한 골프장명 자동완성 제안
4. **스코어 입력 — 4×4 그리드**

```
       │ 타수 │ 퍼팅 │ 버디 │ 포인트│
 비비  │ [  ] │ [  ] │ [  ] │ [  ] │
 에이블│ [  ] │ [  ] │ [  ] │ [  ] │
 에단  │ [  ] │ [  ] │ [  ] │ [  ] │
 제리  │ [  ] │ [  ] │ [  ] │ [  ] │
```

**입력 UX 요구사항:**
- 숫자 키패드 자동 호출 (`inputmode="numeric"`)
- 게임포인트만 `±` 허용 (`inputmode="numeric"` + 부호 버튼)
- 한 칸 입력 후 다음 칸 자동 포커스 이동
- 저장 버튼: 하단 고정, 16칸 모두 채워져야 활성화
- 저장 직후 홈 화면으로 이동 + 토스트 메시지: `"R06 기록 저장 완료 ✨"`

## 7. 디자인 톤 & 비주얼 시스템

**전체 컨셉: K-pop 방송 그래픽 + 스포츠 중계**
- 서바이벌 오디션 프로그램(예: 프로듀스 시리즈, 스트릿우먼파이터) 타이틀 카드 감성
- 스포츠 중계 방송 그래픽(점수 하단 띠, 랭킹 변동 애니메이션) 감성

**컬러 시스템**
- 다크 배경 베이스: `#0F1419` ~ `#1A1F2E` (짙은 네이비/차콜)
- 강조 포인트: 네온 시안 `#00E5FF` 또는 핫 골드 `#FFD54F`
- 카드 배경: 반투명 유리 효과(glassmorphism) 또는 그라디언트
- 플레이어 색상(2장 지정)은 반드시 일관 적용

**타이포그래피**
- 숫자 폰트: **Tabular / Monospace 계열 필수** (예: `JetBrains Mono`, `IBM Plex Mono`, 또는 `Pretendard` with `font-variant-numeric: tabular-nums`)
- 큰 숫자는 굵고 크게, 표 내부 숫자는 정렬된 폭으로
- 한글 본문: `Pretendard` 권장

**모션**
- 순위 변동 시 카드 자동 재정렬 애니메이션
- 1위 카드에 은은한 글로우/반짝임
- 탭 전환 시 부드러운 슬라이드

**이미지/데코**
- 오륜기 5색 원, 골프 클럽 실루엣, 그린/페어웨이 텍스처 최소화하여 포인트로만 사용

## 8. 기술 스택 권장사항

| 영역 | 권장 |
|---|---|
| 프레임워크 | **Next.js 14 (App Router) + TypeScript** |
| 스타일링 | Tailwind CSS + shadcn/ui |
| 차트 | Recharts (꺾은선 + 포인트 마커 지원) |
| 데이터베이스 | **Supabase** (PostgreSQL + 실시간 구독 + Auth) |
| 배포 | Vercel |
| PWA | `next-pwa` 플러그인, manifest.json, 앱 아이콘 512×512, iOS apple-touch-icon |
| 상태관리 | React Query (TanStack Query) |

**실시간 동기화:** 한 명이 라운드를 저장하면 다른 3명 화면에 즉시 반영되도록 Supabase Realtime 구독 적용.

**인증:** 단순화를 위해 단일 공유 비밀번호 or 매직링크(이메일) 방식. 4인 외부 접근 차단.

## 9. 상태 처리 & 엣지 케이스

- **빈 상태:** 라운드 0개일 때 홈 통계 표/카드에 `—` 표시 + "첫 라운드를 입력하세요" CTA
- **부분 입력:** 6라운드만 완료 시 평균은 6라운드 기준 계산, 진행도 `6/10`
- **수정/삭제:** 기록 화면 카드 롱프레스 또는 탭 → 확인 다이얼로그
- **음수 게임포인트:** `-10`처럼 마이너스 부호 명확히 표시, 누적 음수일 때 빨간색 틴트
- **동점 처리:** 평균타수 동률시 → 누적버디 높은 순 → 누적포인트 높은 순으로 tiebreak

## 10. 구현 우선순위 (MVP → 확장)

**Phase 1 (MVP, 최우선 구현):**
1. 입력 화면 (라운드 저장)
2. 홈 화면 누적 통계 표 + 4인 카드
3. 기록 화면 리스트
4. 추이 화면 총타수 그래프 + 표
5. Supabase 연결 + 실시간 동기화
6. PWA 설정 (홈 화면 추가 가능)

**Phase 2 (여유 있을 때):**
- 총 퍼팅수 그래프 + 표
- 수정/삭제 기능
- 시즌 종료 MVP 시상식 페이지
- 라운드 결과 카드 이미지 생성 → 카톡 공유
